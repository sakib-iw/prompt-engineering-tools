Multi-output JSON compare

Paste several JSON documents from repeated model runs. Pick a reference output for shape matching, optionally paste a JSON Schema to validate each document, then inspect structural issues and a side-by-side value matrix for every leaf path.

Every output slot must contain valid JSON. Remove extra slots you are not using. Structural checks compare each output to the reference (keys, array lengths, types).

Schema validation loads Ajv from a CDN when you run Compare (requires network once). If it fails, structural + matrix views still work. Schema uses standard JSON Schema keywords (type, required, properties, etc.).
